Components

Components are the building blocks of the HomeSwift interface. They are reusable UI elements that can be combined to create consistent user experiences across all platforms.

Component Principles

  • Consistency: Components should look and behave consistently across all platforms.
  • Accessibility: Components should be accessible to all users, including those with disabilities.
  • Flexibility: Components should be adaptable to different contexts and use cases.
  • Efficiency: Components should be optimized for performance and ease of implementation.

Buttons

Buttons are used to trigger actions or navigate to new pages. HomeSwift uses three button styles: primary, secondary, and disabled.

Primary Buttons

Primary buttons are used for the main action on a page or in a component. They have high visual prominence and should be used sparingly.

.btn {

.btn-primary {

  color: var(--color-white);
}

.btn-primary:hover {
  background-color: #4b0077;
}

.btn-primary:active {
  background-color: #2d0046;
}

.btn-primary:focus {
  outline: 2px solid var(--color-green-lime);
  outline-offset: 2px;
}

Usage Guidelines

  • Use for primary actions such as "Book Now", "Confirm Payment", or "Submit"
  • Limit to one primary button per screen or section
  • Position in a prominent location, typically at the bottom of forms or dialogs
  • Use clear, action-oriented text (verb + noun when needed)
  • Avoid using for destructive actions (use secondary buttons instead)

Usage Guidelines

  • Use for secondary actions such as "Cancel", "Back", or "View Details"
  • Can be used alongside primary buttons to provide alternative actions
  • Can be used for destructive actions when paired with appropriate warning text
  • Use clear, descriptive text

Disabled Buttons

Disabled buttons indicate that an action is not currently available. They have low visual prominence and cannot be interacted with.

Default

Specifications

Background: Gray (#808080)

Text Color: White (#FFFFFF)

Border: None

Border Radius: 8px

Padding: 12px 24px

Font: Poppins, 16px, Semi-Bold (600)

Opacity: 70%

Cursor: not-allowed

HTML

<button class="btn btn-disabled" disabled>Disabled Button</button>

CSS

.btn-disabled {
  background-color: var(--color-gray);
  color: var(--color-white);
  cursor: not-allowed;
  opacity: 0.7;
}

Usage Guidelines

  • Use to indicate that an action is not currently available
  • Always provide a clear reason why the button is disabled, either through helper text or contextual information
  • Consider using tooltips to explain why a button is disabled
  • Avoid using disabled buttons when possible; consider alternative patterns such as inline validation
Button Accessibility
  • All buttons must have a visible focus state
  • Button text should have a minimum contrast ratio of 4.5:1 against the background
  • Button text should be descriptive and action-oriented
  • Avoid using color alone to convey information
  • Ensure buttons are keyboard accessible
  • Minimum touch target size of 44x44px for mobile interfaces

Cards

Cards are used to group related information and actions. HomeSwift uses three card styles: service cards, provider cards, and booking cards.

Service Cards

Service cards display service categories and options. They provide a quick overview of available services.

🧹

Cleaning

Starting at $25/hr

Specifications

Background: White (#FFFFFF)

Shadow: 0px 2px 8px rgba(0, 0, 0, 0.1)

Border Radius: 10px

Padding: 16px

Width: 200px (responsive)

Icon: Emoji or SVG icon, 36px

Title: Poppins, 18px, Semi-Bold (600)

Price: Poppins, 14px, Regular (400), Gray (#808080)

Button: Primary button, smaller size

HTML

<div class="card service-card">
  <div class="service-card-icon">🧹</div>
  <h3 class="service-card-title">Cleaning</h3>
  <p class="service-card-price">Starting at $25/hr</p>
  <button class="btn btn-primary">View</button>
</div>

Usage Guidelines

  • Use to display service categories on the home screen or service browsing pages
  • Keep content concise and focused on the service name and starting price
  • Use consistent icons across all service cards
  • Ensure the "View" button leads to a detailed service page

Usage Guidelines

  • Use to display booking history and upcoming bookings
  • Include clear status indicators for each booking
  • Provide relevant actions based on the booking status (reschedule, cancel, review)
  • Display the most important booking details prominently
  • Consider adding a "View Details" link for additional information
Card Accessibility
  • Ensure all text has sufficient contrast against the background
  • Use proper heading hierarchy within cards
  • Make sure all interactive elements are keyboard accessible
  • Provide alternative text for images
  • Ensure status indicators use both color and text to convey information

Form Elements

Form elements are used to collect user input. HomeSwift uses consistent form elements across all platforms.

Input Fields

Input fields allow users to enter text data. They are used for various types of information collection.

Please enter a valid value

Specifications

Background: White (#FFFFFF)

Border: 1px solid Gray (#808080)

Border Radius: 8px

Padding: 12px

Font: Poppins, 16px, Regular (400)

Text Color: Midnight Blue (#00305C)

Placeholder Color: Gray (#808080)

Focus State: Green Lime (#37C640) border

Error State: Red (#FF3B30) border with error message

Disabled State: Light Gray (#F5F5F5) background, 70% opacity

HTML

<div class="input-container">
  <label for="input-default">Default Input</label>
  <input type="text" id="input-default" class="input-field" placeholder="Enter text">
</div>

<div class="input-container">
  <label for="input-error">Error Input</label>
  <input type="text" id="input-error" class="input-field input-error" placeholder="Enter text">
  <div class="error-message">Please enter a valid value</div>
</div>

Usage Guidelines

  • Always use labels for input fields
  • Use placeholder text to provide examples or hints, not as a replacement for labels
  • Provide clear error messages when validation fails
  • Group related fields together
  • Use appropriate input types (text, email, tel, etc.)
  • Consider using input masks for formatted data (phone numbers, dates)
