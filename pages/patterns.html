Interaction Patterns

Interaction patterns define how users interact with the HomeSwift platform. These patterns ensure consistency across the user experience and help users accomplish their goals efficiently.

Pattern Principles

HomeSwift interaction patterns are guided by these core principles:

  • Consistency: Patterns should be consistent across the platform to reduce cognitive load.
  • Efficiency: Patterns should help users accomplish tasks with minimal effort.
  • Feedback: Patterns should provide clear feedback on user actions.
  • Accessibility: Patterns should be accessible to all users, regardless of abilities.
  • Predictability: Patterns should behave in ways that users can predict.

Error Handling

Error handling patterns help users recover from errors and understand what went wrong.

Form Validation

Form validation helps users enter valid data and understand validation requirements.

Inline Validation

Please enter a valid email address

Specifications

  • Validate input as the user types or on field blur
  • Show error messages directly below the input field
  • Use red (#FF3B30) for error states
  • Provide clear, actionable error messages
  • Maintain field values when errors occur

Form Submission Errors

Unable to Complete Booking

Please check your payment information and try again.

Specifications

  • Display form-level errors at the top of the form
  • Use a distinct visual treatment for error messages
  • Provide clear instructions on how to resolve the error
  • Maintain form data when errors occur
  • Scroll to the error message automatically

Loading States

Loading states provide feedback to users when content is being loaded or actions are being processed.

Spinners

Spinners indicate that content is loading or an action is being processed.

Button Loading State

Specifications

  • Replace button text with "Loading..." during processing
  • Add a spinner to indicate activity
  • Disable the button during loading
  • Maintain button width to prevent layout shifts
  • Return to original state when loading completes

Content Loading State

Specifications

  • Use centered spinner for full-page or section loading
  • Use primary brand color (#39005C) for spinner
  • Size spinner appropriately for the loading context
  • Consider adding a loading message for longer operations
  • Add a timeout message for extended loading periods

Skeleton Screens

Skeleton screens provide a preview of the content structure while it's loading.

Card Skeleton

Specifications

  • Match the skeleton structure to the actual content
  • Use light gray (#F5F5F5) for skeleton elements
  • Consider adding a subtle animation to indicate loading
  • Maintain consistent spacing and proportions
  • Replace skeleton with actual content progressively
Loading State Accessibility
  • Announce loading states to screen readers using aria-live regions
  • Provide text alternatives for visual loading indicators
  • Ensure loading indicators have sufficient contrast
  • Consider adding progress indicators for longer operations
  • Ensure the UI remains usable during partial loading

Animations

Animations provide feedback, guide attention, and enhance the user experience.

Transition Animations

Transition animations help users understand state changes and navigation.

Button Hover Animation

Specifications

  • Use subtle animations for interactive elements
  • Keep animations short (200-300ms)
  • Use easing functions for natural movement
  • Ensure animations don't interfere with usability
  • Provide a way to reduce motion for users who prefer it

Page Transition

Page transitions use fade and slide effects

Specifications

  • Use consistent transitions for navigation
  • Keep transitions short (300-500ms)
  • Ensure transitions don't delay content access
  • Use directional transitions for hierarchical navigation
  • Respect reduced motion preferences

Feedback Animations

Feedback animations provide visual confirmation of user actions.

Success Animation

Booking Confirmed!

Specifications

  • Use animations to confirm successful actions
  • Keep animations simple and focused
  • Use appropriate colors for different states (success, error, etc.)
  • Ensure animations are visible but not distracting
  • Provide text confirmation alongside animations
Animation Accessibility
  • Respect the prefers-reduced-motion media query
  • Provide alternative static states for users who prefer reduced motion
  • Avoid animations that could trigger vestibular disorders
  • Ensure animations don't interfere with screen reader announcements
  • Keep animations subtle and purposeful

Respecting Reduced Motion Preferences

/* Default animations */
.btn {
  transition: transform 0.2s ease, background-color 0.2s ease;
}

.btn:hover {
  transform: translateY(-2px);
}

/* Reduced motion preference */
@media (prefers-reduced-motion: reduce) {
  .btn {
    transition: background-color 0.2s ease;
  }
  
  .btn:hover {
    transform: none;
  }
}

Input Field Focus

Specifications

  • Highlight form fields on focus
  • Use smooth transitions for focus states
  • Ensure focus states are clearly visible
  • Maintain consistent focus behavior across form elements
  • Provide additional visual cues for active fields
Microinteraction Accessibility
  • Ensure microinteractions don't rely solely on hover states
  • Provide keyboard-accessible alternatives for hover interactions
  • Ensure focus states are clearly visible
  • Use ARIA attributes to communicate state changes to screen readers
  • Test microinteractions with assistive technologies

Content Strategy

Content strategy defines how content is structured, written, and presented across the HomeSwift platform.

  • Use visual cues to indicate content relationships

Tone and Voice

Tone and voice define how HomeSwift communicates with users.

Voice Examples

Friendly and Approachable

"Welcome back! We've found some great cleaning services in your area."

Clear and Direct

"Your booking is confirmed for May 25 at 10:00 AM. Your cleaner will arrive within 15 minutes of the scheduled time."

Helpful and Supportive

"Need to reschedule? No problem! You can change your booking up to 24 hours before the scheduled time without any fees."

Voice Guidelines

  • Friendly but Professional: Warm and approachable without being overly casual
  • Clear and Concise: Direct and easy to understand
  • Helpful and Supportive: Focused on solving user problems
  • Positive and Encouraging: Emphasizing benefits and solutions
  • Culturally Relevant: Appropriate for the Moroccan market

Localization Guidelines

Localization guidelines ensure content is appropriate for different languages and cultures.

Language Support

Supported Languages

  • Arabic (Primary)
  • French (Secondary)
  • English (Tertiary)

Localization Specifications

  • Text Expansion: Allow for 30-50% text expansion in translations
  • Date and Time Formats: Use locale-appropriate formats
  • Currency: Display in Moroccan Dirham (MAD) with appropriate formatting
  • Cultural Considerations: Respect local customs and cultural norms
  • Right-to-Left Support: Full support for Arabic right-to-left layout

RTL Support

/* Base styles */
body {
  font-family: 'Poppins', sans-serif;
  line-height: 1.5;
  color: var(--color-midnight-blue);
}

/* RTL support */
body[dir="rtl"] {
  font-family: 'Poppins', 'Tajawal', sans-serif;
}

.card {
  padding: var(--spacing-4);
}

/* Adjust padding for RTL */
body[dir="rtl"] .card {
  padding-right: var(--spacing-4);
  padding-left: var(--spacing-4);
}
Content Accessibility
  • Use clear, simple language
  • Avoid idioms and colloquialisms that may not translate well
  • Provide text alternatives for non-text content
  • Ensure content is structured logically
  • Use proper heading hierarchy
  • Consider reading level and comprehension
