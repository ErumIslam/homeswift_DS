Design Tokens

Design tokens are the foundational elements of the HomeSwift design system. They define the visual properties that are used consistently across all platforms and components.

What are Design Tokens?

Design tokens are named entities that store visual design attributes. They are used in place of hardcoded values to ensure consistency across the platform. HomeSwift uses design tokens for colors, typography, spacing, shadows, and border styles.

Color Accessibility Note

Green Lime (#37C640) does not meet the 4.5:1 contrast ratio required for normal text on white backgrounds. Use it only for large text (18pt+), UI components, or with additional visual cues.

Warning Yellow (#FFCC00) also does not meet the 4.5:1 contrast ratio for normal text. Always use it with dark text and additional visual cues.

Color Usage Guidelines

  • Use primary colors for main UI elements and brand identity
  • Use secondary colors sparingly to highlight important elements
  • Use status colors consistently to communicate system states
  • Ensure sufficient contrast for all text elements
  • Never use color alone to convey information
  • Consider cultural associations of colors in the Moroccan context

Typography

HomeSwift uses Poppins as its primary font family to create a modern, professional, and readable interface.

Font Family

Poppins is a geometric sans-serif typeface with a clean, modern aesthetic. It provides excellent readability across all screen sizes and devices.

CSS Import

@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;500;600;700&display=swap');

CSS Variables

:root {
  --font-family-primary: 'Poppins', sans-serif;
}

Typography Usage Guidelines

  • Use the type scale consistently to maintain visual hierarchy
  • Maintain minimum font sizes for readability (16px for body text)
  • Use appropriate line spacing (1.5x font size for body text)
  • Limit line length to 80 characters for optimal readability
  • Use font weights to create emphasis and hierarchy
  • Ensure sufficient contrast between text and background
Typography Accessibility

All text must have a minimum contrast ratio of 4.5:1 against its background. Body text should be at least 16px to ensure readability across devices. Text should be resizable up to 200% without loss of content or functionality.

Shadows and Effects

HomeSwift uses shadows to create depth and hierarchy in the interface.

Shadow Scale

Shadows are used to create elevation and focus in the interface. HomeSwift uses three shadow levels for different purposes.

Small Shadow
0px 2px 4px rgba(0, 0, 0, 0.05)
Subtle elevation, buttons, small components
Medium Shadow
0px 2px 8px rgba(0, 0, 0, 0.1)
Cards, containers, dropdown menus
Large Shadow
0px 4px 16px rgba(0, 0, 0, 0.15)
Modals, popovers, elevated content

CSS Variables

:root {
  --shadow-small: 0px 2px 4px rgba(0, 0, 0, 0.05);
  --shadow-medium: 0px 2px 8px rgba(0, 0, 0, 0.1);
  --shadow-large: 0px 4px 16px rgba(0, 0, 0, 0.15);
}

Focus Effects

Focus effects are used to indicate the currently focused element for keyboard navigation.

Focus Outline
0 0 0 2px #37C640
Keyboard focus state for interactive elements

CSS Variables

:root {
  --focus-outline: 0 0 0 2px var(--color-green-lime);
}
Focus Accessibility

Focus indicators must be clearly visible and have sufficient contrast against the background. Never remove focus outlines without providing an alternative focus indicator.

Shadow Usage Guidelines

  • Use shadows consistently to indicate elevation and hierarchy
  • Use small shadows for subtle elevation
  • Use medium shadows for cards and containers
  • Use large shadows for modals and popovers
  • Use focus effects for keyboard navigation
  • Avoid using shadows excessively to prevent visual clutter

  --border-width-emphasis: 2px;
  
  --border-standard: var(--border-width-standard) solid var(--color-gray);
  --border-emphasis: var(--border-width-emphasis) solid var(--color-midnight-blue);
  --border-focus: var(--border-width-emphasis) solid var(--color-green-lime);
}

Border Usage Guidelines

  • Use border radii consistently across similar elements
  • Use small radius for form elements and buttons
  • Use medium radius for cards and containers
  • Use large radius for featured elements
  • Use standard borders for subtle boundaries
  • Use emphasis borders for secondary buttons and selected states
  • Use focus borders for keyboard navigation
