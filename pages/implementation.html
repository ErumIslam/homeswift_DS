Implementation Guidelines

This section provides guidelines for implementing the HomeSwift design system across different platforms and technologies. These guidelines ensure consistency and efficiency in the development process.

Implementation Principles

The implementation of the HomeSwift design system is guided by these core principles:

  • Consistency: Ensure consistent implementation across all platforms and technologies.
  • Efficiency: Optimize implementation for performance and development speed.
  • Maintainability: Create maintainable code that can evolve with the design system.
  • Accessibility: Ensure all implementations meet WCAG 2.1 AA standards.
  • Scalability: Design implementations to scale with the growth of the platform.

Figma Implementation

The HomeSwift design system is implemented in Figma as the primary design tool. This section provides guidelines for working with the Figma implementation.

Figma File Structure

The HomeSwift design system is organized into the following Figma pages:

  • Introduction: Overview and documentation of the design system
  • Design Tokens: Colors, typography, spacing, shadows, and border styles
  • Components: UI components organized by type
  • Patterns: Common UI patterns and layouts
  • Templates: Page templates for common screens
  • Examples: Example screens showing the design system in use

Component Structure

Components in the Figma library are structured using a consistent approach:

Component Organization

  • Main Component: The base component with all variants
  • Variants: Different states and configurations of the component
  • Documentation: Usage guidelines and specifications
  • Examples: Examples of the component in context

Component Naming Convention

Format: [Category]/[Component]/[Variant]

Examples:

  • Buttons/Primary/Default
  • Buttons/Primary/Hover
  • Cards/Service/Default
  • Forms/Input/Default

Auto Layout and Constraints

HomeSwift components use Auto Layout and constraints to ensure responsive behavior:

Auto Layout Guidelines

  • Use Auto Layout for all components
  • Set appropriate padding and spacing using the spacing scale
  • Use "Fill Container" for width when appropriate
  • Use "Hug Contents" for height when appropriate
  • Set appropriate alignment and distribution

JavaScript Behavior

JavaScript Guidelines

  • Implement progressive enhancement

  • Use ARIA attributes for accessibility
  • Optimize for performance

Dropdown Component JavaScript

class Dropdown {
  constructor(element) {
    this.dropdown = element;
    this.button = element.querySelector('.dropdown-button');
    this.menu = element.querySelector('.dropdown-menu');
    this.items = element.querySelectorAll('.dropdown-item');
    
    this.isOpen = false;
    
    this.init();
  }
  
  init() {
    // Set initial ARIA attributes
    this.button.setAttribute('aria-expanded', 'false');
    this.button.setAttribute('aria-controls', this.menu.id);
    
    // Add event listeners
    this.button.addEventListener('click', () => this.toggle());
    this.button.addEventListener('keydown', (e) => this.handleButtonKeydown(e));
    
    this.items.forEach(item => {
      item.addEventListener('click', () => this.selectItem(item));
      item.addEventListener('keydown', (e) => this.handleItemKeydown(e, item));
    });
    
    // Close when clicking outside
    document.addEventListener('click', (e) => {
      if (!this.dropdown.contains(e.target)) {
        this.close();
      }
    });
  }
  
  toggle() {
    if (this.isOpen) {
      this.close();
    } else {
      this.open();
    }
  }
  
  open() {
    this.menu.classList.add('show');
    this.button.setAttribute('aria-expanded', 'true');
    this.isOpen = true;
    
    // Focus first item
    if (this.items.length > 0) {
      this.items[0].focus();
    }
  }
  
  close() {
    this.menu.classList.remove('show');
    this.button.setAttribute('aria-expanded', 'false');
    this.isOpen = false;
  }
  
  selectItem(item) {
    // Handle item selection
    this.button.textContent = item.textContent;
    this.close();
    this.button.focus();
  }
  
  handleButtonKeydown(e) {
    switch (e.key) {
      case 'ArrowDown':
      case 'Down':
        e.preventDefault();
        this.open();
        break;
      case 'Escape':
        e.preventDefault();
        this.close();
        break;
    }
  }
  
  handleItemKeydown(e, item) {
    switch (e.key) {
      case 'ArrowDown':
      case 'Down':
        e.preventDefault();
        if (item.nextElementSibling) {
          item.nextElementSibling.focus();
        }
        break;
      case 'ArrowUp':
      case 'Up':
        e.preventDefault();
        if (item.previousElementSibling) {
          item.previousElementSibling.focus();
        } else {
          this.button.focus();
        }
        break;
      case 'Escape':
        e.preventDefault();
        this.close();
        this.button.focus();
        break;
      case 'Enter':
      case ' ':
        e.preventDefault();
        this.selectItem(item);
        break;
    }
  }
}

// Initialize all dropdowns
document.querySelectorAll('.dropdown').forEach(dropdown => {
  new Dropdown(dropdown);
});

iOS Button Component

// HSButton.swift
import UIKit

class HSButton: UIButton {
    enum Style {
        case primary
        case secondary
        case disabled
    }
    
    private var style: Style = .primary
    
    init(style: Style) {
        super.init(frame: .zero)
        self.style = style
        setupButton()
    }
    
    required init?(coder: NSCoder) {
        super.init(coder: coder)
        setupButton()
    }
    
    private func setupButton() {
        layer.cornerRadius = 8
        titleLabel?.font = UIFont(name: "Poppins-SemiBold", size: 16)
        
        switch style {
        case .primary:
            backgroundColor = UIColor.HomeSwift.plumPurple
            setTitleColor(UIColor.HomeSwift.white, for: .normal)
        case .secondary:
            backgroundColor = UIColor.clear
            setTitleColor(UIColor.HomeSwift.midnightBlue, for: .normal)
            layer.borderWidth = 2
            layer.borderColor = UIColor.HomeSwift.midnightBlue.cgColor
        case .disabled:
            backgroundColor = UIColor.HomeSwift.gray
            setTitleColor(UIColor.HomeSwift.white, for: .normal)
            isEnabled = false
            alpha = 0.7
        }
        
        // Set content insets
        contentEdgeInsets = UIEdgeInsets(top: 12, left: 24, bottom: 12, right: 24)
    }
    
    override var isHighlighted: Bool {
        didSet {
            if style == .primary {
                backgroundColor = isHighlighted ? 
                    UIColor.HomeSwift.plumPurple.darker(by: 20) : 
                    UIColor.HomeSwift.plumPurple
            } else if style == .secondary {
                backgroundColor = isHighlighted ? 
                    UIColor.HomeSwift.midnightBlue.withAlphaComponent(0.1) : 
                    UIColor.clear
            }
        }
    }
}

Android Button Component

// Button.kt
import androidx.compose.foundation.layout.PaddingValues
import androidx.compose.foundation.layout.height
import androidx.compose.foundation.layout.width
import androidx.compose.material.Button
import androidx.compose.material.ButtonDefaults
import androidx.compose.material.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp

enum class ButtonStyle {
    PRIMARY,
    SECONDARY,
    DISABLED
}

@Composable
fun HomeSwiftButton(
    text: String,
    style: ButtonStyle = ButtonStyle.PRIMARY,
    onClick: () -> Unit = {}
) {
    val colors = when (style) {
        ButtonStyle.PRIMARY -> ButtonDefaults.buttonColors(
            backgroundColor = HomeSwiftColors.PlumPurple,
            contentColor = HomeSwiftColors.White
        )
        ButtonStyle.SECONDARY -> ButtonDefaults.buttonColors(
            backgroundColor = HomeSwiftColors.White,
            contentColor = HomeSwiftColors.MidnightBlue
        )
        ButtonStyle.DISABLED -> ButtonDefaults.buttonColors(
            backgroundColor = HomeSwiftColors.Gray,
            contentColor = HomeSwiftColors.White
        )
    }
    
    val isEnabled = style != ButtonStyle.DISABLED
    
    Button(
        onClick = onClick,
        enabled = isEnabled,
        colors = colors,
        contentPadding = PaddingValues(horizontal = 24.dp, vertical = 12.dp),
        modifier = Modifier
            .height(48.dp)
            .width(120.dp)
    ) {
        Text(text = text)
    }
}

Implementation Best Practices

These best practices ensure efficient and consistent implementation of the HomeSwift design system:

General Best Practices

  • Use design tokens consistently across all platforms
  • Implement components according to the specifications
  • Ensure accessibility in all implementations
  • Test implementations across different devices and screen sizes
  • Document implementation details for future reference
  • Maintain version control for all implementation files

Performance Considerations

  • Optimize asset sizes for web and mobile
  • Minimize JavaScript bundle size
  • Use efficient CSS selectors
  • Implement lazy loading for components when appropriate
  • Optimize animations for performance
  • Test performance on low-end devices

Accessibility Implementation

  • Use semantic HTML elements
  • Implement proper ARIA attributes
  • Ensure keyboard navigation works correctly
  • Test with screen readers
  • Implement focus management
  • Support text scaling
Implementation Accessibility

All implementations of the HomeSwift design system must meet WCAG 2.1 AA standards. This includes proper semantic structure, keyboard accessibility, screen reader support, and sufficient color contrast. Test all implementations with assistive technologies to ensure they are accessible to all users.
