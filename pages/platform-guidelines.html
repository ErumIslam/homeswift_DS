Platform Guidelines

HomeSwift is a multi-platform service that operates across mobile apps (iOS and Android), web applications, provider apps, and admin dashboards. These guidelines ensure consistency while respecting the unique characteristics of each platform.

Cross-Platform Principles

While each platform has its own specific guidelines, these core principles apply across all platforms:

  • Consistent Brand Identity: Maintain consistent brand colors, typography, and visual language across all platforms.
  • Platform-Appropriate Interactions: Adapt interaction patterns to match platform conventions and user expectations.
  • Shared Component Library: Use the same core components with platform-specific adaptations.
  • Accessibility First: Ensure all platforms meet WCAG 2.1 AA standards.
  • Responsive Design: Adapt layouts and components to different screen sizes and orientations.

Mobile App Guidelines

The HomeSwift mobile apps for iOS and Android are the primary touchpoint for customers seeking home services.

The iOS app follows Apple's Human Interface Guidelines while maintaining HomeSwift's brand identity.

iOS-Specific Components

iOS Navigation Bar

Follows iOS action sheet patterns with HomeSwift colors

iOS Design Specifications

  • Navigation: iOS-native navigation bar with HomeSwift branding
  • Tab Bar: Standard iOS tab bar with HomeSwift icons
  • Buttons: Follow iOS button styles with HomeSwift colors
  • Typography: Poppins font (not SF Pro) to maintain brand consistency
  • Gestures: Support standard iOS gestures (swipe, pinch, tap)
  • Dark Mode: Full support for iOS dark mode

iOS Technical Requirements

  • Support latest iOS version and one version back
  • Optimize for iPhone and iPad
  • Support all iPhone screen sizes
  • Implement VoiceOver accessibility
  • Support Dynamic Type for text scaling
  • Implement proper keyboard handling

Cross-Platform Consistency

Maintaining consistency across platforms while respecting platform-specific conventions is essential for a cohesive user experience.

Consistency Checklist

  • Visual Identity:
    • Use consistent brand colors across all platforms
    • Maintain Poppins as the primary font family
    • Use consistent iconography with platform-appropriate styling
  • Component Behavior:
    • Adapt component behavior to platform conventions
    • Maintain consistent naming and terminology
    • Ensure similar components have similar functionality
  • Content:
    • Use consistent tone and voice across platforms
    • Maintain consistent terminology for features and actions
    • Adapt content length to platform constraints
  • Workflows:
    • Maintain consistent user flows across platforms
    • Adapt interaction patterns to platform conventions
    • Ensure feature parity where appropriate
